## Lab 6 — Numeric Types

### Concept
Python has three built-in numeric types:

| Type | Name | Example | Use for |
|------|------|---------|---------|
| `int` | integer | `5`, `-3`, `1000000` | whole numbers, counts |
| `float` | floating-point | `3.14`, `-0.5`, `2.0` | decimals, measurements |
| `complex` | complex number | `2 + 3j` | advanced math, signals |

You'll use `int` and `float` constantly; `complex` rarely (mostly in signal processing).

**Arithmetic operators** — the tools for doing math:

| Operator | Name | Example | Result |
|----------|------|---------|--------|
| `+` | addition | `7 + 2` | `9` |
| `-` | subtraction | `7 - 2` | `5` |
| `*` | multiplication | `7 * 2` | `14` |
| `/` | **true division** (always float) | `7 / 2` | `3.5` |
| `//` | **floor division** (drops remainder) | `7 // 2` | `3` |
| `%` | **modulo** (remainder) | `7 % 2` | `1` |
| `**` | exponent (power) | `2 ** 10` | `1024` |

Three that trip up beginners:
- **`/` always gives a float**, even `4 / 2` gives `2.0`, not `2`.
- **`//` floors** toward negative infinity: `7 // 2` is `3`.
- **`%` (modulo)** gives the remainder — hugely useful for "is this even?" (`n % 2 == 0`) and cycling.

**Float precision warning:** Computers store decimals in binary, so `0.1 + 0.2` gives `0.30000000000000004`. This is normal in every language — never test floats for exact equality.

### Example
Create `numbers.py`:

```python
"""numbers.py — Demonstrates Python's numeric types and operators."""

a = 7
b = 2

print(f"{a} + {b}  = {a + b}")     # 9
print(f"{a} - {b}  = {a - b}")     # 5
print(f"{a} * {b}  = {a * b}")     # 14
print(f"{a} / {b}  = {a / b}")     # 3.5   (true division -> float)
print(f"{a} // {b} = {a // b}")    # 3     (floor division)
print(f"{a} % {b}  = {a % b}")     # 1     (remainder)
print(f"{b} ** 10  = {b ** 10}")   # 1024  (2 to the power 10)

# Types
print(type(5))       # <class 'int'>
print(type(3.14))    # <class 'float'>
print(type(7 / 2))   # <class 'float'>  -- note: division makes a float!

# The famous float precision quirk
print(0.1 + 0.2)     # 0.30000000000000004
```

Output:

```text
7 + 2  = 9
7 - 2  = 5
7 * 2  = 14
7 / 2  = 3.5
7 // 2 = 3
7 % 2  = 1
2 ** 10  = 1024
<class 'int'>
<class 'float'>
<class 'float'>
0.30000000000000004
```

## Lab 7 — Strings Basics

### Concept
A **string** is a sequence of characters (text), written between quotes — single `'...'` or double `"..."` (both work identically):

```python
greeting = "Hello AI"
```

**Strings are immutable** — once created, you cannot change a character in place. Operations that "modify" a string actually create a *new* string.

**Indexing** — access a single character by its position, in square brackets. Python counts **from 0**:

```text
 H   e   l   l   o
 0   1   2   3   4      <- positive indices
-5  -4  -3  -2  -1      <- negative indices (count from the end)
```

- `s[0]` → first character
- `s[-1]` → last character (negative indices count backward)

**Slicing** — extract a *range* of characters with `s[start:stop]`. The `start` is included, the `stop` is **excluded**:

```python
s[1:4]   # characters at index 1, 2, 3 (NOT 4)
```

You can add a step: `s[start:stop:step]`. A step of `-1` reverses the string: `s[::-1]`.

### Example
Create `strings.py`:

```python
"""strings.py — Indexing and slicing strings."""

s = "Hello"

# Indexing (single characters)
print(s[0])    # H  (first)
print(s[1])    # e
print(s[-1])   # o  (last)
print(s[-2])   # l  (second from last)

# Slicing (ranges) -- stop index is EXCLUDED
print(s[0:3])  # Hel   (indices 0,1,2)
print(s[1:4])  # ell   (indices 1,2,3)
print(s[:3])   # Hel   (start defaults to 0)
print(s[2:])   # llo   (stop defaults to end)
print(s[::-1]) # olleH (reversed)

# Length of a string
print(len(s))  # 5

# Immutability: this would ERROR ->  s[0] = "J"
# Instead, build a NEW string:
new_s = "J" + s[1:]
print(new_s)   # Jello
```

Output:

```text
H
e
o
l
Hel
ell
Hel
llo
olleH
5
Jello
```

## Lab 8 — String Methods

### Concept
Strings come with built-in **methods** — functions attached to the string that you call with a dot: `s.method()`. Because strings are immutable, these methods **return a new string** (they never change the original).

Most useful methods:

| Method | What it does | Example | Result |
|--------|--------------|---------|--------|
| `.upper()` | uppercase everything | `"hi".upper()` | `"HI"` |
| `.lower()` | lowercase everything | `"HI".lower()` | `"hi"` |
| `.strip()` | remove leading/trailing whitespace | `"  hi  ".strip()` | `"hi"` |
| `.replace(a, b)` | replace all `a` with `b` | `"aaa".replace("a","b")` | `"bbb"` |
| `.split(sep)` | break into a list | `"a,b,c".split(",")` | `["a","b","c"]` |
| `.title()` | Capitalize Each Word | `"hi ai".title()` | `"Hi Ai"` |
| `.startswith(x)` | test the beginning | `"hi".startswith("h")` | `True` |
| `.find(x)` | index of first match (or -1) | `"hello".find("l")` | `2` |

You can **chain** methods left to right: `"  Hello  ".strip().lower()` first strips, then lowercases.

### Example
Create `string_methods.py`:

```python
"""string_methods.py — Common string methods and chaining."""

messy = "  Hello, AI  "

print(messy.upper())          # '  HELLO, AI  '
print(messy.lower())          # '  hello, ai  '
print(messy.strip())          # 'Hello, AI'  (whitespace removed)
print(messy.replace(",", ""))  # '  Hello AI  '  (comma removed)

# Chaining: strip -> lower -> replace the comma
clean = messy.strip().lower().replace(",", "")
print(clean)                   # 'hello ai'

# Splitting text into a list of words
sentence = "learn python and ai"
words = sentence.split(" ")
print(words)                   # ['learn', 'python', 'and', 'ai']
print(len(words), "words")     # 4 words

# Tests
print("hello".startswith("he"))  # True
print("hello".find("l"))         # 2
```

Output:

```text
  HELLO, AI  
  hello, ai  
Hello, AI
  Hello AI  
hello ai
['learn', 'python', 'and', 'ai']
4 words
True
2
```

### Lab Practice

3. Split the sentence `"learn python and ai"` into a list of words and count them.
4. Check whether `"artificial"` starts with `"art"`.
5. Prove the original string is unchanged after calling `.upper()` on it (print it again).

### Solution
```python

```

## Lab 9 — f-strings & Formatting

### Concept
**f-strings** (formatted string literals) are the modern, readable way to build text from variables. Put an `f` before the opening quote and place expressions inside `{}`:

```python
name = "AI"
print(f"Hello, {name}!")   # -> Hello, AI!
```

You can put **any expression** inside the braces, not just variable names:

```python
print(f"2 + 2 = {2 + 2}")            # 2 + 2 = 4
print(f"Upper: {name.upper()}")      # Upper: AI
```

**Format specifiers** control how a value is displayed, using a colon inside the braces: `{value:spec}`.

| Spec | Meaning | Example | Output |
|------|---------|---------|--------|
| `.2f` | 2 decimal places (float) | `f"{3.14159:.2f}"` | `3.14` |
| `.0f` | 0 decimals (rounded) | `f"{3.7:.0f}"` | `4` |
| `,` | thousands separator | `f"{1000000:,}"` | `1,000,000` |
| `%` | percentage | `f"{0.25:.1%}"` | `25.0%` |
| `>10` | right-align in 10 chars | `f"{'hi':>10}"` | `'        hi'` |
| `<10` | left-align in 10 chars | `f"{'hi':<10}"` | `'hi        '` |
| `^10` | center in 10 chars | `f"{'hi':^10}"` | `'    hi    '` |
| `08.2f` | pad with zeros, 2 decimals | `f"{3.14:08.2f}"` | `00003.14` |

You can also add `=` for quick debugging: `f"{x=}"` prints `x=5`.

### Example
Create `formatting.py`:

```python
"""formatting.py — f-strings and format specifiers."""

pi = 3.14159
price = 1234.5
ratio = 0.847
name = "AI"

# Basic substitution
print(f"Hello, {name}!")

# Expressions inside braces
print(f"2 + 2 = {2 + 2}")
print(f"Uppercase name: {name.upper()}")

# Rounding floats
print(f"Pi to 2 decimals: {pi:.2f}")     # 3.14
print(f"Pi to 4 decimals: {pi:.4f}")     # 3.1416

# Thousands separator and currency
print(f"Price: ${price:,.2f}")           # $1,234.50

# Percentages
print(f"Success rate: {ratio:.1%}")      # 84.7%

# Alignment (useful for tables)
print(f"|{'left':<10}|{'center':^10}|{'right':>10}|")

# Debug form
x = 42
print(f"{x=}")                           # x=42
```

Output:

```text
Hello, AI!
2 + 2 = 4
Uppercase name: AI
Pi to 2 decimals: 3.14
Pi to 4 decimals: 3.1416
Price: $1,234.50
Success rate: 84.7%
|left      |  center  |     right|
x=42
```

### Lab Practice
1. Create `pi = 3.14159` and print it rounded to 2 decimals.
2. Print `1234567` with thousands separators (`1,234,567`).
3. Given `ratio = 0.925`, print it as a percentage with 1 decimal (`92.5%`).

### Solution


## Lab 10 — Booleans & Truthiness

### Concept
A **boolean** is a value that is either `True` or `False` (note the capital letters). Booleans are the foundation of decision-making in programs — every `if` statement ultimately checks a boolean.

Booleans usually come from **comparisons**:

```python
5 > 3      # True
5 == 3     # False
```

**Truthiness** — Python can treat *any* value as a boolean in a condition. Some values are "falsy" (act like `False`); everything else is "truthy" (acts like `True`).

**Falsy values** (there are only a few — memorize them):
- `False`
- `0` and `0.0` (zero of any numeric type)
- `""` (empty string)
- `[]`, `{}`, `()`, `set()` (empty collections)
- `None`

**Everything else is truthy** — non-zero numbers, non-empty strings, non-empty lists, etc.

You can check truthiness explicitly with the `bool()` function:

```python
bool("")      # False (empty string)
bool("hi")    # True  (non-empty string)
bool(0)       # False
bool(42)      # True
bool([])      # False (empty list)
bool([1, 2])  # True  (non-empty list)
```

Why this matters: it lets you write clean checks like `if name:` (meaning "if name is not empty") instead of `if name != "":`.

### Example
Create `booleans.py`:

```python
"""booleans.py — Booleans and truthiness."""

# Booleans from comparisons
print(5 > 3)      # True
print(5 == 3)     # False
print(5 != 3)     # True
print(2 <= 2)     # True

# The two boolean values
print(type(True))   # <class 'bool'>

# Truthiness: bool() reveals how Python sees a value in a condition
print(bool(""))      # False -> empty string is falsy
print(bool("hi"))    # True  -> non-empty string is truthy
print(bool(0))       # False -> zero is falsy
print(bool(42))      # True  -> non-zero is truthy
print(bool([]))      # False -> empty list is falsy
print(bool([1]))     # True  -> non-empty list is truthy
print(bool(None))    # False -> None is falsy

# Practical use: truthiness in an if-statement
name = ""
if name:
    print(f"Hello, {name}")
else:
    print("No name provided")   # this runs, because "" is falsy
```

Output:

```text
True
False
True
True
<class 'bool'>
False
True
False
True
False
True
False
No name provided
```

### Lab Practice
1. Print the results of `1 < 2`, `3 == 3.0`, and `"a" != "b"`.
2. Use `bool()` on each of these and predict the result first: `""`, `"0"`, `0`, `[1]`, `[]`, `None`.
3. (Tricky!) What is `bool("0")` — the string `"0"`? Explain why.
4. Write an `if`/`else` that prints `"Empty!"` when a list variable is empty and `"Has items"` otherwise, using truthiness (no `len()`).

### Solution
```python
print(1 < 2)        # True
print(3 == 3.0)     # True  (int and float compare by value)
print("a" != "b")   # True

print(bool(""))     # False
print(bool("0"))    # True  <- IMPORTANT: it's a NON-empty string!
print(bool(0))      # False (the number zero)
print(bool([1]))    # True
print(bool([]))     # False
print(bool(None))   # False

items = []
if items:
    print("Has items")
else:
    print("Empty!")     # runs, because [] is falsy
```

**Why `bool("0")` is `True`:** Truthiness of a string depends only on whether it has characters — not on what those characters are. `"0"` contains one character, so it's non-empty and therefore truthy. Only the completely empty string `""` is falsy.
